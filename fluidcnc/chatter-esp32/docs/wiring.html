ESP32 CNC Chatter System – Wiring Guide

Clean, practical wiring for the ESP32 chatter detector (MPU‑6050 + INMP441 + ACS712) and UART into a BTT Octopus Pro running grblHAL. This page is designed to be “follow-along” with a multimeter.

Quick safety rules read first

  • ESP32 GPIO is 3.3V logic. Never feed 5V into an ESP32 GPIO pin.
  • Use a common ground between ESP32 and Octopus Pro UART.
  • For any “voltage sensor”, ensure the ADC pin never exceeds 3.3V.
  • If you’re working inside mains/VFD wiring: disconnect power and be safe.

ESP32 ↔ Sensors (what goes where)

ACS712 current sensor (spindle load) — ACS712T ELC‑30A (30A)

ACS712 pinESP32Notes
VCC5V (VIN)Most ACS712 modules are powered from 5V.
GNDGNDCommon ground.
OUTGPIO 34 (ADC1)ADC pin must stay ≤ 3.3V. Add a divider if your module output can exceed 3.3V.
Important: The ACS712 output is centered around VCC/2. Many modules can swing up near ~4.5V when powered at 5V. Protect the ESP32 ADC with a simple divider (example below) if you’re not sure.
For the ACS712T ELC‑30A (30A), typical sensitivity is ~66mV/A at 5V supply. With the 10k/20k divider below (0.666×), the ESP32 ADC effectively sees ~44mV/A.

Optional: “Voltage sensor” (measuring 12V/24V/48V safely)

If you mean a common “0–25V voltage sensor module” (a resistor divider board): it’s just a divider feeding an ADC.

Voltage sensor moduleESP32Notes
V+ / VINMeasure point (+)Example: 24V PSU positive.
GNDESP32 GNDCommon ground required.
S / OUTGPIO 34 (or another ADC1 pin)Ensure divider keeps OUT ≤ 3.3V.
Do NOT measure VFD motor output with an ESP32 voltage sensor module. Measure only safe DC rails (12/24/48V) unless you use proper isolation.
If your goal is “VFD/spindle telemetry” (recommended approaches):
  • Modbus/RS‑485 (best): many VFDs can report frequency, RPM, output current, DC bus voltage, faults. This is the safest and most useful data for automation.
  • Analog monitor output (often labeled AM/FM + ACM): typically 0–10V or 0–5V representing frequency or current. This can be read by an ADC only if you scale to ≤3.3V and follow the VFD manual.
  • Speed pulse/tach output: some VFDs expose a digital “frequency output” (open collector). This is ideal for RPM feedback using a GPIO + pullup (no analog scaling needed).

This project uses ESP32 UART2 so you don’t interfere with the USB programming port.

Baud rate should match grblHAL (commonly 115200). If you see garbage characters, baud mismatch is the first thing to check.

VFD Modbus RS‑485 Telemetry (Optional but Recommended)

Most 1.5kW–2.2kW spindle kits from Amazon/AliExpress include a Huanyang or clone VFD with RS‑485 Modbus support. This gives you the richest telemetry: actual frequency, output current, DC bus voltage, and fault codes—directly from the VFD.

Hardware needed

  • RS‑485 transceiver module (MAX485, SP3485, or similar). Ideally isolated (ADM2587E) for noisy VFD environments.
  • 3 wires from ESP32 to RS‑485 module, 2–3 wires from RS‑485 module to VFD.

Common Huanyang VFD parameters (set on VFD panel):
  • PD163 = 1 (communication mode)
  • PD164 = 1 (slave address = 1)
  • PD165 = 1 (baud 9600) or 2 (19200)
The firmware defaults to slave address 1 @ 9600 baud. Adjust in vfd_modbus.h if your VFD differs.

Enabling in firmware

In main.cpp, uncomment these lines in setup():

vfd.begin();
vfd.setSpindleConfig(400.0f, 24000.0f, 4);  // maxFreq, maxRPM, poles

After flashing, the TFT and web UI will show VFD data (frequency, current, DC bus, faults).

Troubleshooting RS‑485: If you get no data, double-check A↔RS+ and B↔RS-. Some modules swap labeling. Try swapping A/B if you see 100% failures in the diagnostics panel.

Voltage divider examples (protecting GPIO34 ADC)

ESP32 ADC pins (like GPIO34) should be kept ≤ 3.3V. A divider is two resistors:

Vout = Vin × (R2 / (R1 + R2))

Limit ACS712 OUT (just in case) R1=10k, R2=20k Scales by 0.666 (5V → 3.33V max-ish). Add a small cap (e.g. 0.01µF) from ADC to GND for noise.
